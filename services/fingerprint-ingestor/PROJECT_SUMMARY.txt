PROJECT OVERVIEW
================================================================================
A high-throughput HTTP fingerprint ingestion service for Synora (Advertisement
Recognition as a Service). Receives fingerprint batches from TV SDK devices and
publishes them to Apache Kafka for downstream processing.

TARGET SPECIFICATIONS
- Throughput: 500,000 requests/second
- Language: Go 1.22
- Framework: valyala/fasthttp (10x faster than net/http)
- Message Queue: Apache Kafka
- Cache/State: Redis
- Metrics: Prometheus
- Status: PRODUCTION READY

================================================================================
ARCHITECTURE & DESIGN
================================================================================

REQUEST FLOW
1. Client sends POST /v1/fingerprints with X-API-Key
2. Auth middleware validates API key (Redis + local cache)
3. Request body parsed and validated (JSON, batch size, format)
4. Rate limiter checks per-key limit (Redis sliding window)
5. Each event validated: device_id, fingerprint_hash, timestamp
6. GeoIP filter checks IP address (MaxMind GeoIP2 database)
7. Valid events published to Kafka (async, fire-and-forget)
8. 202 Accepted response returned immediately
9. Prometheus metrics recorded (counters, histograms)

KAFKA MESSAGE SCHEMA
{
  "device_id": "64-char-hex",
  "fingerprint_hash": "64-char-hex",
  "timestamp_utc": 1712748932000,
  "manufacturer": "LG",
  "model": "OLED55C3",
  "ip_address": "203.0.113.42",
  "ingested_at": 1712748932123
}

COMPONENTS
- fasthttp Server: HTTP request handling
- Kafka Producer: Async message publishing
- Redis Client: Rate limiting, API key storage
- GeoIP Filter: IP validation using MaxMind
- Prometheus Metrics: Observability
- zap Logger: Structured logging
- JWT/API Key: Authentication

EXTERNAL DEPENDENCIES
- Apache Kafka (message queue)
- Redis (rate limiting, state)
- MaxMind GeoIP2 (IP filtering)
